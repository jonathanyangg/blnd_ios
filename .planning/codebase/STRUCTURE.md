## Directory Purposes

**App:**
- Purpose: Application lifecycle and root state injection
- Contains: @main entry point
- Key files: `blndApp.swift` — initializes AuthState + OnboardingState, injects into environment

**Config:**
- Purpose: Runtime configuration, API base URL, Keychain storage, Supabase settings
- Contains: Constants and utility enums
- Key files: `APIConfig.swift` (base URL), `KeychainManager.swift` (Security framework), `SupabaseConfig.swift` (storage bucket)

**Models:**
- Purpose: Codable structs matching backend API schemas
- Contains: Request/response models, value objects (Genre, CastMember)
- Key files: Each file covers one domain (Auth, Movies, Tracking, Friends, Groups, Import)
- Pattern: CodingKeys for snake_case ↔ camelCase mapping

**Networking:**
- Purpose: HTTP transport, API abstractions, error handling
- Contains: APIClient singleton + domain-specific enum APIs + avatar upload
- Key files: `APIClient.swift` (core HTTP, Bearer token, error parsing); domain enums call `APIClient.shared`
- Pattern: All API calls go through APIClient; domain enums add type safety/convenience

**State:**
- Purpose: Shared observable application state
- Contains: @Observable classes for auth, tab selection, onboarding progress
- Key files: `AuthState.swift` (login/logout/currentUser), `TabState.swift` (selected tab), `OnboardingState.swift` (credentials/genres/ratings cache)
- Pattern: @Observable allows reactive updates; injected via .environment()

**Theme:**
- Purpose: Centralized design system tokens
- Contains: Color palette, spacing scale, corner radius presets, gradients
- Key files: `AppTheme.swift` — all color/size decisions reference this file

**Views:**
- Purpose: SwiftUI view hierarchy organized by feature
- Contains: Screens, sheets, local components
- Key files: One main view per feature + supporting child views; Shared/ for cross-feature components
- Organization: Auth (onboarding steps), Home (feed + search + movie detail), Friends (list + add), Groups (list + detail + create), Profile (user data + settings), Import (external data)

**Shared:**
- Purpose: Reusable UI components used across features
- Contains: Buttons, cards, search bars, avatars, sheets, gesture modifiers
- Key files: ComponentName.swift per component (AppButton, AvatarView, MovieCard, etc.)
- Pattern: Lightweight Views with simple parameters, no complex business logic

## Naming Conventions

**Files:**
- SwiftUI Views: PascalCase.swift (e.g., `HomeView.swift`, `AppButton.swift`)
- Configuration: PascalCase.swift (e.g., `APIConfig.swift`, `KeychainManager.swift`)
- State files: PascalCase + State suffix (e.g., `AuthState.swift`, `TabState.swift`)
- Models: PascalCase + Models suffix (e.g., `AuthModels.swift`, `MovieModels.swift`)
- API enums: PascalCase + API suffix (e.g., `AuthAPI.swift`, `MoviesAPI.swift`)

**Directories:**
- Feature directories: PascalCase (e.g., `Home/`, `Friends/`, `Groups/`, `Profile/`, `Auth/`)
- Shared utilities: Shared/ (all cross-feature components)
- Functional directories: lowercase (Config, Networking, Models, State, Theme)

**Code Identifiers:**
- Views: PascalCase (e.g., `HomeView`, `AppButton`)
- Functions: camelCase (e.g., `loadWatched()`, `fetchMovie()`)
- Variables: camelCase (e.g., `selectedTab`, `isLoading`)
- Enums/Types: PascalCase (e.g., `HomeTab`, `APIError`)
- Constants: camelCase, grouped in types (e.g., `AppTheme.background`)

## Where to Add New Code

**New Feature:**
- Primary code: Create new feature directory under `Views/` (e.g., `Views/Discovery/`)
- API endpoints: Add static functions to existing API enum in `Networking/`, or create new enum if domain-specific
- Models: Add Codable structs to existing file in `Models/`, or create new file if large domain
- State: Add new @Observable class to `State/` only if shared across multiple features
- Tests: Co-locate near views (no dedicated test directory yet)

**New Component/Module:**
- Shared across features: Place in `Views/Shared/ComponentName.swift`
- Feature-specific: Place in `Views/FeatureName/ComponentName.swift`
- Reusable logic: Extract to `Networking/` or `State/` as appropriate

**Utilities:**
- Shared helpers (like SwipeBackGesture): `Views/Shared/`
- API/network utilities: `Networking/`
- Configuration/secrets: `Config/`
- Theme tokens: `Theme/AppTheme.swift`

## Special Directories

**Assets.xcassets:**
- Purpose: App icons, colors, images (generated/managed by Xcode)
- Generated: Yes (by Xcode)
- Committed: Yes

**Views/Shared:**
- Purpose: Reusable UI components
- Generated: No
- Committed: Yes
- Pattern: Import across all features

**Views/Auth:**
- Purpose: Onboarding flow only (not login post-signup)
- Generated: No
- Committed: Yes
- Note: Removed from navigation stack after successful signup

**Networking:**
- Purpose: No domain logic — only request/response handling
- Generated: No (AvatarUploader.swift added for Supabase uploads)
- Committed: Yes
- Pattern: APIClient is singleton, domain APIs are enums
